ajari aku cara pakai hermes yang uda jalan di vps sumopod, untuk mengerjakan COMOT ini

Untuk memanfaatkan Hermes yang sudah berjalan di VPS Sumopod untuk membangun MVP COMOT, kita bisa menggunakannya sebagai asisten coding otomatis (AI Developer) langsung di server. Hermes sangat bertenaga untuk code generation, membuat struktur repositori, hingga debugging berbasis instruksi teks (prompt).

Berikut adalah panduan langkah demi langkah cara mengoperasikan dan memanfaatkannya untuk mengeksekusi proyek COMOT.

1. Memastikan Akses ke Hermes di VPS

Biasanya, Hermes di VPS dijalankan menggunakan salah satu dari dua cara: via CLI (Terminal) atau via Web UI (seperti Open WebUI atau Ollama API).

Opsi A: Jika Hermes Berjalan via Web UI / Port API

Jika VPS-mu sudah membuka port untuk akses web (misal port 3000 atau 11434 untuk Ollama):

Buka browser dan masuk ke alamat IP VPS kamu (contoh: http://IP_VPS_KAMU:3000).

Pilih model Hermes (misalnya Hermes-3-Llama-3.1 atau versi yang kamu instal).

Opsi B: Jika Akses Lewat SSH Terminal (CLI)

Jika kamu masuk ke VPS via SSH, kamu bisa langsung memanggil Hermes menggunakan runner seperti Ollama:

Bash
ssh user@ip_vps_sumopod
ollama run hermes3 # (atau sesuaikan dengan nama model yang kamu install)


Setelah masuk ke prompt >>>, kamu bisa langsung mengobrol dengan Hermes.

2. Strategi Eksekusi Proyek COMOT Menggunakan Hermes

Jangan langsung menyuruh Hermes membuat "seluruh aplikasi COMOT" sekaligus karena hasilnya akan terpotong. Kita akan menyuapinya menggunakan PRD.md yang sudah kita buat sebelumnya secara bertahap.

Langkah 1: Inisialisasi Struktur Proyek

Minta Hermes membuatkan struktur folder dasar untuk MVP COMOT (kita asumsikan menggunakan stack modern dan cepat seperti Node.js/Express untuk Backend dan React/Next.js untuk Frontend, atau Python jika kamu fokus ke AI).

Prompt ke Hermes:

"Saya ingin membuat MVP untuk COMOT berdasarkan PRD yang ada. Tolong buatkan struktur folder boilerplate untuk aplikasi web berbasis Node.js (Express) untuk backend dan React untuk frontend. Berikan juga file package.json dasarnya."

Langkah 2: Membuat Integrasi API Qhomemart (Mocking)

Karena MVP ini membutuhkan integrasi API Purchasing Qhomemart, minta Hermes membuatkan skrip integrasi atau mock API-nya terlebih dahulu.

Prompt ke Hermes:

"Buatlah sebuah modul backend Express di file controllers/qhomemartController.js untuk menghubungkan API purchasing Qhomemart. Karena ini MVP, buatkan mock data yang mensimulasikan status order material seperti 'Pending', 'In Transit', dan 'Delivered'."

Langkah 3: Membuat Fitur Pencatatan Lapangan (Database & Route)

Pekerja lapangan butuh antarmuka sederhana untuk mencatat material yang masuk/keluar. Minta Hermes membuatkan skema database (misal menggunakan PostgreSQL atau MongoDB) dan route-nya.

Prompt ke Hermes:

"Buat skema database Mongoose (MongoDB) untuk MaterialLog yang mencatat nama proyek, nama material, kuantitas, status (masuk/keluar), dan timestamp. Berikan juga API route POST /api/logs untuk mencatat data dari lapangan."

3. Alur Kerja Otomatis: Menghubungkan Hermes ke File Sistem VPS

Jika kamu ingin Hermes bekerja lebih efisien langsung membaca dan menulis file di VPS, kamu bisa memanfaatkan Aider atau Continue.dev yang dihubungkan ke model Hermes di VPS-mu.

Namun, jika kamu menggunakan Hermes secara manual lewat terminal/web:

Salin Code dari Hermes: Biarkan Hermes men-generate kode di layar.

Gunakan nano atau vim: Di terminal VPS, buat filenya.

Bash
nano backend/controllers/qhomemartController.js


Paste dan Save: Tempel kode dari Hermes, lalu simpan (Ctrl+O, lalu Ctrl+X di nano).

Python
from crewai import Agent, Task, Crew, Process
from langchain_community.llms import Ollama

# 1. Inisialisasi Hermes dari Ollama lokal
hermes_llm = Ollama(model="hermes3", base_url="http://localhost:11434")

# 2. Definisikan Agen sesuai kriteria gambar (Multi-Agent Workflow)
coordinator = Agent(
    role='Project Coordinator',
    goal='Mengoordinasikan kebutuhan logistik proyek rumah Rp5 Miliar',
    backstory='Kamu adalah manajer proyek senior yang ahli melihat inefisiensi lapangan.',
    verbose=True,
    llm=hermes_llm
)

procurement_officer = Agent(
    role='Qhomemart Procurement Specialist',
    goal='Mencocokkan kebutuhan material dengan ketersediaan stok di Qhomemart API',
    backstory='Kamu ahli dalam negosiasi harga, pelacakan pesanan, dan pemenuhan barang tepat waktu.',
    verbose=True,
    llm=hermes_llm
)

# 3. Definisikan Tugas & Ekspektasi Output (Mencakup Reasoning & Structured Output)
task_analisis_keterlambatan = Task(
    description='Analisis dampak jika pengiriman semen instan terlambat 3 hari pada proyek saat ini.',
    expected_output='Format JSON dengan key: "status_risiko", "estimasi_kerugian_rupiah", "rekomendasi_tindakan".',
    agent=coordinator
)

# 4. Jalankan Proses Komunikasi Antar Agent (Workflow & Orchestration)
comot_crew = Crew(
    agents=[coordinator, procurement_officer],
    tasks=[task_analisis_keterlambatan],
    process=Process.sequential # Agen berkomunikasi berurutan
)

result = comot_crew.kickoff()
print(result)

from crewai import Agent, Task, Crew, Process
from langchain_community.llms import Ollama

# 1. Inisialisasi Hermes dari Ollama lokal
hermes_llm = Ollama(model="hermes3", base_url="http://localhost:11434")

# 2. Definisikan Agen sesuai kriteria gambar (Multi-Agent Workflow)
coordinator = Agent(
    role='Project Coordinator',
    goal='Mengoordinasikan kebutuhan logistik proyek rumah Rp5 Miliar',
    backstory='Kamu adalah manajer proyek senior yang ahli melihat inefisiensi lapangan.',
    verbose=True,
    llm=hermes_llm
)

procurement_officer = Agent(
    role='Qhomemart Procurement Specialist',
    goal='Mencocokkan kebutuhan material dengan ketersediaan stok di Qhomemart API',
    backstory='Kamu ahli dalam negosiasi harga, pelacakan pesanan, dan pemenuhan barang tepat waktu.',
    verbose=True,
    llm=hermes_llm
)

# 3. Definisikan Tugas & Ekspektasi Output (Mencakup Reasoning & Structured Output)
task_analisis_keterlambatan = Task(
    description='Analisis dampak jika pengiriman semen instan terlambat 3 hari pada proyek saat ini.',
    expected_output='Format JSON dengan key: "status_risiko", "estimasi_kerugian_rupiah", "rekomendasi_tindakan".',
    agent=coordinator
)

# 4. Jalankan Proses Komunikasi Antar Agent (Workflow & Orchestration)
comot_crew = Crew(
    agents=[coordinator, procurement_officer],
    tasks=[task_analisis_keterlambatan],
    process=Process.sequential # Agen berkomunikasi berurutan
)

result = comot_crew.kickoff()
print(result)
